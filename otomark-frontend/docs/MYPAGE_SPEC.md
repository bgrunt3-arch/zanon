# マイページ 仕様書

## 概要

Orbit アプリのマイページ。ログイン中の Spotify ユーザー情報と、選択中の推し5人を表示・管理する。

---

## 現状の実装

### 表示内容

| セクション | 内容 |
|-----------|------|
| ヘッダー | 「マイページ」タイトル |
| プロフィール | アバター画像 / フォールバック（名前頭2文字）、「Spotifyユーザー」ラベル、表示名 |
| エラー時 | メッセージ + 再ログインボタン |
| 推し5人 | セクションタイトル「現在選んでいる5人」、説明文、5人グリッド |
| アクション | 「アーティストを入れ替える」ボタン → オンボーディングへ |

### アクセス制御

- トークンなし → `/login` へリダイレクト
- 推しが5人未満 → `/onboarding` へリダイレクト
- 15秒タイムアウトでエラーメッセージ表示

### データソース

- `fetchMe(token)` … Spotify Me API
- `getSelectedArtists()` … localStorage の推し5人

## 仕様の詰め（提案）

### 1. プロフィールセクション

| 項目 | 現状 | 提案 |
|------|------|------|
| アバター | 68×68px、画像 or フォールバック | `aspect-ratio: 1/1` で読み込み前後のサイズ変動を防止 |
| 表示名 | `display_name` そのまま | 空の場合は「ユーザー」などフォールバック |
| ラベル | 「Spotifyユーザー」固定 | そのまま |
| リンク | なし | 将来: Spotify プロフィールへのリンク（任意） |

### 2. 推し5人セクション

| 項目 | 現状 | 提案 |
|------|------|------|
| カード | 画像 + 名前のみ | 各カードをタップでアーティスト詳細ページへ遷移（`/artists/[id]`） |
| 並び順 | オンボーディングで選んだ順 | そのまま（並び替えは将来対応） |
| 空状態 | 5人未満はオンボーディングへ | そのまま |
| スケルトン | 5枚グリッド | そのまま |

### 3. アクション

| 項目 | 現状 | 提案 |
|------|------|------|
| 入れ替え | オンボーディングへ遷移 | そのまま |
| ログアウト | なし | 追加: 「ログアウト」ボタン（トークン削除 → `/login`） |
| 設定 | なし | 将来: モックモード切替など |

### 4. ローディング・エラー

| 項目 | 現状 | 提案 |
|------|------|------|
| ローディング | プロフィール + グリッドスケルトン | そのまま |
| 401 | 再ログインへ | そのまま |
| タイムアウト | 15秒 | そのまま |
| ネットワークエラー | メッセージ + 再ログイン | リトライボタン追加を検討 |

### 5. レイアウト・UX

| 項目 | 提案 |
|------|------|
| スクロール | コンテンツ量に応じて可動 |
| タイトル | ヘッダー固定 or スクロールで追従（現状維持） |
| ボタン配置 | 入れ替えボタンは下部に固定、ログアウトはセクション末尾に |

## 実装優先度

1. **必須**: アーティストカードをタップで詳細ページへ遷移
2. **推奨**: ログアウトボタン追加
3. **推奨**: アバターの `aspect-ratio` でレイアウトシフト防止
4. **将来**: 並び替え、設定、プロフィール編集

---

## 画面遷移

```
/mypage
  ├─ トークンなし → /login
  ├─ 推し5人未満 → /onboarding
  ├─ アーティストカードタップ → /artists/[artistId]
  ├─ 入れ替えボタン → /onboarding
  └─ ログアウト → /login（トークン削除）
```

---

## モックモード

- `NEXT_PUBLIC_MOCK_MODE=true` または `localStorage.orbit.mockMode=1` 時
- `fetchMe` はモックデータを返す
- マイページはそのまま表示可能
